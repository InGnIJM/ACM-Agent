## 设计哲学

> "算法的本质是穷举，关键是知道如何聪明地穷举。" — labuladong

原 6.4 设计的问题：
1. **画像维度太粗** — 只有 tag_proficiency 一个维度，缺少结构化评估
2. **计算方法缺失** — LLM 主观打分，缺乏可量化、可复现的公式
3. **训练规划缺乏科学依据** — 难度曲线公式过于简单，没有间隔重复、知识依赖
4. **没有借鉴成熟算法教学体系** — 缺少"框架思维"和"套路模板"维度

本设计引入 **6 维用户画像 + 4 阶段训练模型 + 知识依赖图谱**，所有数值计算走纯 Python，LLM 只做语义分析和个性化编排。

### 2.2 D1: 知识覆盖率（Coverage）

衡量用户涉猎的算法知识广度。

```python
def calc_coverage(user_tags: set[str], all_tags: set[str]) -> float:
    """
    user_tags: 用户 AC 过的题目涉及的 tags_normalized
    all_tags: 题库中所有 tags_normalized
    """
    if not all_tags:
        return 0.0
    return len(user_tags & all_tags) / len(all_tags)
```

**扩展 — 分类覆盖率**（用于雷达图）:
```python
def calc_category_coverage(user_tags, taxonomy):
    """返回每个 category 的覆盖率"""
    result = {}
    for cat, subcats in taxonomy.items():
        cat_tags = {t for sub in subcats for t in sub.topics}
        covered = user_tags & cat_tags
        result[cat] = len(covered) / len(cat_tags) if cat_tags else 0
    return result
```

### 2.3 D2: 知识掌握度（Proficiency）

每个 tag 独立计算，综合 AC 率、难度、题量、时效 4 个因子。

```python
def calc_proficiency(
    ac_count: int,        # 该 tag 下 AC 题数
    total_count: int,     # 该 tag 下总提交数
    avg_difficulty: float, # AC 题目平均难度 (1~10)
    days_since_last: int,  # 距最后一次 AC 的天数
) -> float:
    """
    proficiency = w_ac × ac_factor + w_diff × diff_factor + w_vol × vol_factor + w_rec × recency_factor
    权重: ac=0.40, difficulty=0.30, volume=0.20, recency=0.10
    """
    # AC 率因子: sigmoid 映射，0.5 AC 率 → 0.5 分
    ac_rate = ac_count / max(total_count, 1)
    ac_factor = 1 / (1 + math.exp(-10 * (ac_rate - 0.5)))  # sigmoid

    # 难度因子: 线性归一化
    diff_factor = min(avg_difficulty / 10.0, 1.0)

    # 题量因子: 对数衰减，30 题趋近饱和
    vol_factor = min(math.log(1 + ac_count) / math.log(31), 1.0)

    # 时效因子: 指数衰减，30 天半衰期
    recency_factor = math.exp(-0.023 * days_since_last)  # ln(2)/30 ≈ 0.023

    proficiency = (0.40 * ac_factor +
                   0.30 * diff_factor +
                   0.20 * vol_factor +
                   0.10 * recency_factor)
    return round(proficiency, 3)
```

**关键参数说明**:
| 参数 | 值 | 设计意图 |
|------|---|---------|
| sigmoid 斜率 10 | ac_rate 0.3→0.27, 0.5→0.50, 0.7→0.73 | 区分度在 30%~70% AC 率最敏感 |
| log(31) 归一化 | 10 题→0.65, 20 题→0.82, 30 题→0.97 | 鼓励多刷但不无限刷 |
| 30 天半衰期 | 15 天→0.71, 30 天→0.50, 60 天→0.25 | 遗忘曲线近似 |

### 2.4 D3: 难度天花板（Ceiling）

衡量用户能稳定 AC 的最高难度。

```python
def calc_ceiling(records: list[dict]) -> float:
    """
    取近 90 天内，AC 题目难度的 P90（第 90 百分位）。
    不用 max，因为 max 容易被偶然 AC 的水题/运气题干扰。
    不用 avg，因为 avg 会被大量简单题拉低。
    """
    ac_difficulties = [
        r['difficulty_normalized'] for r in records
        if r['verdict'] == 'OK' and r['days_ago'] <= 90
    ]
    if len(ac_difficulties) < 5:
        return 0.0  # 数据不足
    return np.percentile(ac_difficulties, 90)
```

**分段映射为等级**:
| ceiling 范围 | 等级 | 描述 |
|-------------|------|------|
| 1~3 | 入门 | 能做基础模拟、简单数据结构 |
| 4~5 | 初级 | 能做经典算法模板题 |
| 6~7 | 中级 | 能做中等综合题 |
| 8~9 | 高级 | 能做省赛/区域赛中档题 |
| 10 | 精英 | 能做区域赛/World Finals 难度 |

### 2.5 D4: 解题效率（Efficiency）

衡量"一次做对"的能力，而非暴力提交。

```python
def calc_efficiency(records: list[dict]) -> float:
    """
    efficiency = first_ac_rate × 0.6 + (1 - avg_retries/max_retries) × 0.4

    first_ac_rate: 首次提交即 AC 的比例
    avg_retries: 平均每题从首次提交到 AC 的尝试次数
    max_retries: 归一化上限（设为 5）
    """
    problem_attempts = defaultdict(list)
    for r in records:
        problem_attempts[r['problem_id']].append(r['verdict'])

    first_ac_count = 0
    total_retries = 0
    for pid, verdicts in problem_attempts.items():
        if 'OK' in verdicts:
            first_ac_idx = verdicts.index('OK')
            if first_ac_idx == 0:
                first_ac_count += 1
            total_retries += first_ac_idx

    total_problems = len(problem_attempts)
    if total_problems == 0:
        return 0.0

    first_ac_rate = first_ac_count / total_problems
    avg_retries = total_retries / total_problems
    retry_factor = 1 - min(avg_retries / 5.0, 1.0)

    return round(0.6 * first_ac_rate + 0.4 * retry_factor, 3)
```

### 2.6 D5: 学习风格（Learning Style）

基于行为模式自动分类，不用 LLM 主观判断。

```python
def classify_style(
    total_solved: int,
    unique_tags: int,
    avg_proficiency: float,
    top3_tag_concentration: float,  # 前 3 tag 占总 AC 的比例
    avg_difficulty: float,
) -> str:
    """
    基于 4 个阈值的规则分类器:

    1. 题海型 (grinder): total_solved > 200 AND unique_tags > 15
       → 刷得多、涉猎广，但不一定深

    2. 精研型 (deep_diver): total_solved < 100 AND avg_proficiency > 0.7
       → 刷得少但每题吃透，AC 率高、效率高

    3. 偏科型 (specialist): top3_tag_concentration > 0.6
       → 60% 以上的题集中在 3 个 tag

    4. 均衡型 (balanced): 以上都不满足
       → 正常分布
    """
    if total_solved > 200 and unique_tags > 15:
        return "grinder"
    if total_solved < 100 and avg_proficiency > 0.7:
        return "deep_diver"
    if top3_tag_concentration > 0.6:
        return "specialist"
    return "balanced"
```

**style_bonus 映射**（用于综合评分）:

### 2.7 D6: 趋势动量（Momentum）

衡量用户近期进步/退步趋势。

```python
def calc_momentum(daily_stats: list[dict], window: int = 30) -> float:
    """
    对近 window 天的每日 AC 数做线性回归，取斜率归一化到 [-1, 1]。

    momentum > 0: 进步中（推荐加大难度）
    momentum < 0: 退步/倦怠（推荐降低难度 + 复习）
    momentum ≈ 0: 平台期（推荐换方向突破）
    """
    if len(daily_stats) < 7:
        return 0.0

    recent = daily_stats[-window:]
    y = [s['ac_count'] for s in recent]
    x = list(range(len(y)))

    # 简单线性回归: slope = Σ(xi - x̄)(yi - ȳ) / Σ(xi - x̄)²
    x_mean = sum(x) / len(x)
    y_mean = sum(y) / len(y)
    numerator = sum((xi - x_mean) * (yi - y_mean) for xi, yi in zip(x, y))
    denominator = sum((xi - x_mean) ** 2 for xi in x)

    if denominator == 0:
        return 0.0

    slope = numerator / denominator
    # 归一化: 假设日均 AC 变化 0.5 题为显著，映射到 [-1, 1]
    return round(max(-1, min(1, slope / 0.5)), 3)
```

## 三、4 阶段训练规划模型

### 3.1 训练阶段设计（借鉴 labuladong 学习路径）

labuladong 的核心教学理念：**先框架后细节，先二叉树培养递归思维，再回溯/动规/贪心**。

融合 ACM 竞赛特点，设计 4 阶段递进训练模型：

```
阶段 1: 模板巩固期 (Template Consolidation)   — 1~2 周
阶段 2: 专题突破期 (Topic Breakthrough)       — 2~4 周
阶段 3: 综合提升期 (Integrated Practice)      — 2~4 周
阶段 4: 竞赛模拟期 (Contest Simulation)       — 持续
```

| 阶段 | 目标 | 题目特征 | 难度范围 | 每日题量 | labuladong 对应 |
|------|------|---------|---------|---------|----------------|
| 模板巩固 | 补齐知识盲区，强化模板 | 单一 tag 模板题 | ceiling-3 ~ ceiling-1 | 5~8 | 基础系列 + 框架模板 |
| 专题突破 | 攻克弱点 tag | 弱点 tag 变种题 | ceiling-1 ~ ceiling+1 | 4~6 | 数据结构系列 + 动规系列 |
| 综合提升 | 跨 tag 综合运用 | 多 tag 融合题 | ceiling ~ ceiling+2 | 3~5 | 高频面试系列 |
| 竞赛模拟 | 限时实战 | 真题/模拟赛 | 全难度 | 3~5 题/场 | — |

### 3.2 阶段自动判定

```python
def determine_phase(profile: UserProfile) -> str:
    """根据用户画像自动判定当前应处于哪个训练阶段"""

    # 阶段 1: 覆盖率低 OR 多个基础 tag proficiency < 0.4
    weak_basics = sum(1 for t in BASIC_TAGS if profile.tag_proficiency.get(t, 0) < 0.4)
    if profile.coverage < 0.3 or weak_basics >= 3:
        return "template_consolidation"

    # 阶段 2: 有明确弱点 tag (top 3 weakness gap > 0.5)
    has_clear_weakness = any(w['gap'] > 0.5 for w in profile.weaknesses[:3])
    if has_clear_weakness and profile.ceiling < 7:
        return "topic_breakthrough"

    # 阶段 3: ceiling >= 6 且效率 >= 0.5
    if profile.ceiling >= 6 and profile.efficiency >= 0.5:
        return "integrated_practice"

    # 阶段 4: ceiling >= 8 且综合评分 >= 0.7
    if profile.ceiling >= 8 and profile.overall_score >= 0.7:
        return "contest_simulation"

    # 默认: 专题突破
    return "topic_breakthrough"
```

### 3.3 训练靶向确定（Target Selection）

```python
def select_targets(profile: UserProfile, phase: str) -> dict:
    """
    确定训练靶向: 主攻 tag + 巩固 tag + 探索 tag

    返回:
    {
        "primary": [{"tag": str, "reason": str, "priority": int}],    # 2~3 个主攻
        "secondary": [{"tag": str, "reason": str}],                   # 1~2 个巩固
        "explore": [{"tag": str, "reason": str}]                      # 0~1 个探索
    }
    """
    targets = {"primary": [], "secondary": [], "explore": []}

    if phase == "template_consolidation":
        # 主攻: 基础 tag 中 proficiency 最低的 3 个
        basic_tags = sorted(BASIC_TAGS,
                           key=lambda t: profile.tag_proficiency.get(t, 0))
        for tag in basic_tags[:3]:
            targets["primary"].append({
                "tag": tag,
                "reason": f"基础 tag，当前掌握度 {profile.tag_proficiency.get(tag, 0):.2f}",
                "priority": 1
            })

    elif phase == "topic_breakthrough":
        # 主攻: weakness 中 gap 最大且满足依赖的 2~3 个
        for w in profile.weaknesses[:5]:
            tag = w["tag"]
            deps_ok = all(
                profile.tag_proficiency.get(dep, 0) >= 0.5
                for dep in DEPENDENCY_GRAPH.get(tag, [])
            )
            if deps_ok:
                targets["primary"].append({
                    "tag": tag,
                    "reason": f"薄弱点，gap={w['gap']:.2f}",
                    "priority": w.get("priority", 1)
                })
            if len(targets["primary"]) >= 3:
                break

        # 巩固: radar 中 0.5~0.65 的边缘 tag
        for tag, score in profile.tag_proficiency.items():
            if 0.5 <= score <= 0.65 and tag not in [t["tag"] for t in targets["primary"]]:
                targets["secondary"].append({"tag": tag, "reason": f"边缘掌握度 {score:.2f}"})
            if len(targets["secondary"]) >= 2:
                break

    elif phase == "integrated_practice":
        # 主攻: 跨 category 的综合 tag 组合
        weak_cats = [cat for cat, cov in profile.category_coverage.items() if cov < 0.5]
        for cat in weak_cats[:2]:
            targets["primary"].append({
                "tag": cat,
                "reason": f"分类覆盖率低 ({profile.category_coverage[cat]:.2f})",
                "priority": 1
            })

    elif phase == "contest_simulation":
        # 主攻: 全面提升，重点补齐短板
        for w in profile.weaknesses[:2]:
            targets["primary"].append({"tag": w["tag"], "reason": "赛前查漏补缺", "priority": 1})
        # 探索: 用户未接触的高级 tag
        for tag in ADVANCED_TAGS:
            if profile.tag_proficiency.get(tag, 0) == 0:
                targets["explore"].append({"tag": tag, "reason": "未接触的高级知识点"})
                break

    return targets
```

### 3.4 难度曲线计算（v2）

原版公式过于简单，v2 引入 **ZPD（最近发展区）理论**：

```python
def calc_difficulty_curve(
    phase: str,
    ceiling: float,
    weak_avg_difficulty: float,
    momentum: float,
    plan_days: int = 7,
) -> list[float]:
    """
    基于 ZPD 理论的难度曲线:
    - 起点: 舒适区 (当前水平 - 1)
    - 峰值: 最近发展区 (当前水平 + 1~2)
    - 根据 momentum 动态调整攀升速度

    ZPD 理论: 学习发生在舒适区和恐慌区之间的"最近发展区"
    """
    if phase == "template_consolidation":
        # 模板巩固: 低起点、缓攀升
        start = max(1.0, ceiling - 3.0)
        peak = max(start + 1, ceiling - 1.0)
        ramp_days = plan_days - 1

    elif phase == "topic_breakthrough":
        # 专题突破: 中起点、中攀升
        start = max(1.0, weak_avg_difficulty - 1.0)
        peak = min(10.0, weak_avg_difficulty + 2.0)
        ramp_days = plan_days - 2

    elif phase == "integrated_practice":
        # 综合提升: 高起点、陡攀升
        start = max(1.0, ceiling - 1.0)
        peak = min(10.0, ceiling + 2.0)
        ramp_days = plan_days - 2

    elif phase == "contest_simulation":
        # 竞赛模拟: 难度波动，模拟真实比赛
        start = max(1.0, ceiling - 2.0)
        peak = min(10.0, ceiling + 1.0)
        ramp_days = plan_days - 1

    # momentum 调整: 进步快→加大难度跨度，退步→缩小跨度
    momentum_factor = 1.0 + 0.3 * momentum  # range: 0.7 ~ 1.3
    effective_peak = start + (peak - start) * momentum_factor
    effective_peak = min(10.0, max(start + 0.5, effective_peak))

    # 生成曲线: 线性攀升 + 最后一天回稳
    curve = []
    for day in range(plan_days):
        if day < ramp_days:
            t = day / max(ramp_days - 1, 1)
            diff = start + (effective_peak - start) * t
        else:
            diff = effective_peak - 1.0  # 最后一天回稳
        curve.append(round(max(1.0, min(10.0, diff)), 1))

    return curve
```

### 3.5 题目召回与打分（v2）

原版 3 路召回保留，打分公式增强：

```python
def score_problem(
    problem: dict,
    target_tags: list[str],
    target_difficulty: float,
    user_solved: set[str],
    user_proficiency: dict[str, float],
) -> float:
    """
    综合打分 = 标签匹配(0.30) + 难度匹配(0.25) + 向量相似(0.15)
             + 新颖性(0.15) + 依赖满足(0.15)

    新增:
    - 新颖性: 用户未接触过的 tag 加分
    - 依赖满足: 前置知识已掌握的题加分
    """
    # 标签匹配
    tag_overlap = len(set(problem['tags_normalized']) & set(target_tags))
    tag_score = tag_overlap / max(len(target_tags), 1)

    # 难度匹配: 高斯核，σ=1.5
    diff = problem['difficulty_normalized']
    diff_score = math.exp(-0.5 * ((diff - target_difficulty) / 1.5) ** 2)

    # 向量相似 (由 PGVector 返回)
    vector_score = problem.get('vector_similarity', 0.5)

    # 新颖性: 涉及用户 proficiency < 0.3 的 tag 越多越新颖
    novel_tags = [t for t in problem['tags_normalized']
                  if user_proficiency.get(t, 0) < 0.3]
    novelty_score = min(len(novel_tags) / max(len(problem['tags_normalized']), 1), 1.0)

    # 依赖满足: 所有前置 tag 的平均 proficiency
    deps = []
    for tag in problem['tags_normalized']:
        deps.extend(DEPENDENCY_GRAPH.get(tag, []))
    if deps:
        dep_score = sum(user_proficiency.get(d, 0) for d in deps) / len(deps)
    else:
        dep_score = 1.0  # 无前置依赖

    total = (0.30 * tag_score +
             0.25 * diff_score +
             0.15 * vector_score +
             0.15 * novelty_score +
             0.15 * dep_score)

    # 已 AC 惩罚
    if problem['id'] in user_solved:
        total *= 0.1

    return round(total, 4)
```

### 3.6 间隔重复调度（Spaced Repetition）

借鉴 Anki 的 SM-2 算法，对弱点 tag 做间隔复习：

```python
def schedule_review(tag: str, history: list[dict]) -> dict:
    """
    SM-2 变体: 根据历史表现决定下次复习时间

    history: [{date, quality: 0~5}] quality 参考 SM-2 标准
        0 = 完全不会
        1 = 错误，但看到答案后记起
        2 = 错误，但答案很熟悉
        3 = 正确，但很费力
        4 = 正确，略有犹豫
        5 = 完美一次过

    返回: {next_review_date, interval_days, ease_factor}
    """
    if not history:
        return {"next_review_date": today(), "interval_days": 1, "ease_factor": 2.5}

    last = history[-1]
    prev_interval = len(history)  # 简化: 用历史次数作为间隔基数
    ease = 2.5

    if last['quality'] >= 3:
        # 答对: 拉长间隔
        if prev_interval == 1:
            interval = 6
        else:
            interval = round(prev_interval * ease)
        ease = max(1.3, ease + 0.1 - (5 - last['quality']) * (0.08 + (5 - last['quality']) * 0.02))
    else:
        # 答错: 重置间隔
        interval = 1
        ease = max(1.3, ease - 0.2)

    return {
        "next_review_date": today() + timedelta(days=interval),
        "interval_days": interval,
        "ease_factor": round(ease, 2),
    }
```

**训练规划中的应用**: 每周计划中，至少 20% 的题目用于复习弱点 tag（间隔到期的）。
